## 产品概述

针对 `/v1/courseTables/studentCourseList` 接口进行性能优化，创建全新的 `/v2/courseTables/studentCourseList` 接口，独立于现有 v1 版本。

## 核心需求

1. **新增 V2 接口**：路径从 `/v1` 变更为 `/v2/courseTables/studentCourseList`
2. **Service 层独立**：新建 Service 文件编写优化逻辑，不影响原有 v1 接口
3. **入参出参不变**：保持与 v1 接口一致的请求参数和响应结构
4. **Native Query 优化**：使用 `entityManager.createNativeQuery` 动态查询
5. **表关联优化**：利用冗余字段将表关联从 5 张减少到 3 张

## 核心优化策略

### 关键发现：冗余字段利用

通过分析 `CourseTable.java` 实体，发现已有冗余字段：

```java
// CourseTable.java 冗余字段
private String courseName;      // 冗余了课程名称
private String collegeName;     // 冗余了学院名称  
private String teacherName;     // 冗余了教师姓名
```

**这意味着可以完全避免 JOIN tb_course 和 tb_college 表！**

### 表关联优化对比

| 方案 | 涉及表 | JOIN 数量 | 性能 |
| --- | --- | --- | --- |
| 原 JPA 方案 | course_table + course + college + group + group_member + course_category | 5 张表 | 高复杂度 |
| **优化方案** | course_table + group + group_member | **3 张表** | **极优** |


### EntityManager.createNativeQuery 动态查询

**优势**：

1. 动态条件构建（根据 userId/openId）
2. 避免静态 SQL 的条件分支
3. 精确控制查询字段
4. 直接映射到 DTO

**核心查询设计（仅 3 张表）**：

```sql
SELECT DISTINCT
    ct.id as courseTableId,
    ct.course_id as courseId,
    ct.course_name as courseName,        -- 冗余字段，无需 JOIN course
    ct.college_name as collegeName,       -- 冗余字段，无需 JOIN college
    ct.teacher_name as teacherName,       -- 冗余字段
    cc.id as courseCategoryId,
    cc.course_category_name as courseCategoryName,
    g.id as groupId,
    g.group_name as groupName,
    g.group_no as groupNo,
    g.sort as groupSort,
    g.source as groupSource
FROM tb_course_table ct
INNER JOIN tb_group g ON ct.group_id = g.id
INNER JOIN tb_group_member gm ON g.id = gm.group_id
LEFT JOIN tb_course_category cc ON ct.course_category_id = cc.id
WHERE gm.student_id = :userId            -- 或 gm.open_id = :openId
  AND ct.school_year = :schoolYear
  AND ct.term = :term
  AND g.group_status = 0                 -- GroupStatus.NORMAL
  AND gm.group_member_status = 0         -- GroupMemberStatus.NORMAL
ORDER BY cc.id, ct.course_id
```

**补充查询（班级人数）**：

```sql
SELECT 
    group_id,
    COUNT(*) as memberCount
FROM tb_group_member
WHERE group_id IN (:groupIds)
  AND group_member_status = 0
GROUP BY group_id
```

## 关键代码结构

### StudentCourseListV2Service 核心方法

```java
@Service
public class StudentCourseListV2Service {
    
    @PersistenceContext
    private EntityManager entityManager;
    
    public ResponseEntity<List<CourseStudentResource>> getStudentCourseList(
            String userId, String schoolYear, Integer term, String openId) {
        
        // 1. 构建动态 SQL
        StringBuilder sql = new StringBuilder();
        sql.append("SELECT DISTINCT ");
        sql.append("ct.id as courseTableId, ");
        sql.append("ct.course_id as courseId, ");
        sql.append("ct.course_name as courseName, ");
        sql.append("ct.college_name as collegeName, ");
        sql.append("ct.teacher_name as teacherName, ");
        sql.append("cc.id as courseCategoryId, ");
        sql.append("cc.course_category_name as courseCategoryName, ");
        sql.append("g.id as groupId, ");
        sql.append("g.group_name as groupName, ");
        sql.append("g.group_no as groupNo, ");
        sql.append("g.sort as groupSort ");
        sql.append("FROM tb_course_table ct ");
        sql.append("INNER JOIN tb_group g ON ct.group_id = g.id ");
        sql.append("INNER JOIN tb_group_member gm ON g.id = gm.group_id ");
        sql.append("LEFT JOIN tb_course_category cc ON ct.course_category_id = cc.id ");
        
        // 2. 动态 WHERE 条件
        List<String> conditions = new ArrayList<>();
        if (StringUtils.hasText(userId)) {
            conditions.add("gm.student_id = :userId");
        } else {
            conditions.add("gm.open_id = :openId");
        }
        conditions.add("ct.school_year = :schoolYear");
        conditions.add("ct.term = :term");
        conditions.add("g.group_status = 0");
        conditions.add("gm.group_member_status = 0");
        
        sql.append("WHERE ").append(String.join(" AND ", conditions));
        sql.append(" ORDER BY cc.id, ct.course_id");
        
        // 3. 创建查询并设置参数
        Query query = entityManager.createNativeQuery(sql.toString());
        if (StringUtils.hasText(userId)) {
            query.setParameter("userId", userId);
        } else {
            query.setParameter("openId", openId);
        }
        query.setParameter("schoolYear", schoolYear);
        query.setParameter("term", term);
        
        // 4. 执行查询并映射结果
        List<Object[]> results = query.getResultList();
        
        // 5. 转换为 CourseStudentResource
        List<CourseStudentResource> resources = convertToResource(results);
        
        // 6. 补充班级人数（批量查询）
        fillGroupMemberCount(resources);
        
        return new ResponseEntity<>(resources, HttpStatus.OK);
    }
}
```
